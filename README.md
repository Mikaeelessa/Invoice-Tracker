# Trade Payments Dashboard — Stakeholder View

A read-only invoice tracking dashboard hosted on GitHub Pages.

---

## 🚀 Setup Guide (5 minutes)

### Step 1 — Create a GitHub account
Go to [github.com](https://github.com) and sign up for a free account if you don't have one.

### Step 2 — Create a new repository
1. Click the **+** button (top right) → **New repository**
2. Name it: `payments-dashboard` (or anything you like)
3. Set visibility to **Private** (recommended for internal financial data) or Public
4. Click **Create repository**

### Step 3 — Upload the files
1. Click **Add file** → **Upload files**
2. Upload **both** files from this folder:
   - `index.html`
   - `README.md`
3. Click **Commit changes**

### Step 4 — Enable GitHub Pages
1. Go to your repo → **Settings** (top menu)
2. Scroll down to **Pages** (left sidebar)
3. Under **Source**, select **Deploy from a branch**
4. Set branch to **main**, folder to **/ (root)**
5. Click **Save**

### Step 5 — Get your link
After ~60 seconds, GitHub will show your live URL:
```
https://YOUR-USERNAME.github.io/payments-dashboard/
```
Copy this link and share it with your stakeholders.

## 🔄 How to update the dashboard data

When you make changes in your editor dashboard (mark invoices paid, add new ones, etc.), update the viewer by:

1. In your **editor dashboard**, click **⬇ Export CSV** — this gives you a record of current data
2. Open `index.html` in a text editor (Notepad, VS Code, etc.)
3. Find the line that starts with `const DATA = [`
4. Replace the entire array with your updated data
5. Update the `DATA_DATE` line to today's date: `const DATA_DATE = '2026-04-15';`
6. Go back to GitHub → your repo → click `index.html` → click the **pencil (edit) icon**
7. Paste in the updated file content → click **Commit changes**

The live URL updates automatically within ~30 seconds.

## 🔒 Privacy note

- **Private repo**: Only people you explicitly invite (Settings → Collaborators) can see the repo code — but the **GitHub Pages URL is still publicly accessible** to anyone with the link.
- If you need the URL to be truly private (login required), upgrade to GitHub Pro ($4/month) which supports private Pages, or use Netlify with password protection instead.
- For internal stakeholders who should not share the link externally, simply treat the URL as confidential.
